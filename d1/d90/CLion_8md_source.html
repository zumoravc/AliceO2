17 ## AliBuild integration
18 
19 The difficulty here lies with the fact that the O2 software needs to be configures with cmake using a number of variables that point to the installation of its dependencies.
20 These instructions have to be carried out only once.
21 
22 1. Build O2 using alibuild as usual
23 2. Open the log file that should be in ~/alice/sw/BUILD/O2-latest/log
24 3. Near the top of the file identify the line starting with `+ cmake`.
25 4. Copy the arguments starting from `-DFairRoot` till the end of the command.
26 5. Paste it in an editor
27 6. The annoying part starts : replace all references to a version by "latest", e.g. :
28  * `-DFairRoot_DIR=/home/local/alice/sw/slc7_x86-64/FairRoot/alice-dev-20170711-1` -> `-DFairRoot_DIR=/home/local/alice/sw/slc7_x86-64/FairRoot/latest`
29  * `-DPROTOBUF_PROTOC_EXECUTABLE=/home/local/alice/sw/slc7_x86-64/protobuf/v3.0.2-1/bin/protoc` -> `-DPROTOBUF_PROTOC_EXECUTABLE=/home/local/alice/sw/slc7_x86-64/protobuf/latest/bin/protoc`
30 7. In CLion, open Settings and go to the tab "Build,Execution,Deployment".
31 8. Click `CMake` and expand the "CMake options". Copy the string you substituted.
32 9. Save
33 10. Reopen clion from a terminal in which you will have loaded the environment : `alienv load O2/latest`
