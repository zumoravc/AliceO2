5 # Digitizer Workflow
6 
7 This is a short documention for the DPL-DigitizerWorkflow example
8 
9 # Status/Description of implementation
10 
11 At present, the `o2-sim-digitizer-workflow` executable is a demonstrator of
12 how we intend to do initiate and handle the processing of hits, coming from detector simulation.
13 
14 The `o2-sim-digitizer-workflow` currently demonstrates the transformation of hits into TPC digits using
15 realistic bunch crossing and collision sampling. We are also able to overlay hits from background and signal hit inputs.
16 
17 The main components of the `o2-sim-digitizer-workflow` are
18 
19 * The SimReader device:
20  - reading/analysing the given hit files
21  - performing the bunch crossing sampling and collision composition (stored in a collision context)
22  - initiating digitization/processing by communicating the collision context to processing devices
23 
24 * The TPC digitizier device:
25  - producing digits in continuous time for a given sector
26  - at present writes (or forwards) these digits in units of TPC drift times
27 
28 The `o2-sim-digitizer-workflow` executable is already somewhat configurable, both in terms of
29 workflow/topology options as well as individual device options. Some help is available
30 via
31 ```
32 o2-sim-digitizer-workflow --help
33 ```
34 Other features are demonstrated in he following section.
35 
36 # Feature example/Usage
37 
38 Let's assume we have a background hit file `o2sim_bg.root` generated
39 by the O2 simulation with
40 ```
41 o2-sim -n 20 -g SOMEBACKGROUNDEVENTGENERATOR -m [detectors] -o o2sim_bg.root
42 ```
43 
44 Similar for a signal file `o2sim_sg.root`
45 ```
46 o2-sim -n 50 -g SOMESIGNALEVENTGENERATOR -m [detectors] -o o2sim_sg.root
47 ```
48 
49 1. **How can I digitize all sectors for the given background event?**
50  ```
51  o2-sim-digitizer-workflow -b --simFile o2sim_bg.root
52  ```
53  This will run as many TPC digitizer processors as there are logical CPU cores on your machine in parallel.
54  (Note that depending on your available memory, this might cause problems as the digitization needs lots of memory; It might be safer to start with a small number of workers as indicated under point 3.).
55 
56 2. **How can I only digitize sectors TPC sectors 1 + 2 for the given background event?**
57  ```
58  o2-sim-digitizer-workflow -b --tpc-sectors=1,2 --simFile o2sim_bg.root
59  ```
60 
61 3. **How can I digitize sectors 1-8 using only 2 TPC digitizer devices?**
62  ```
63  o2-sim-digitizer-workflow -b --tpc-lanes=2 --tpc-sectors=1,2,3,4,5,6,7,8 --simFile o2sim_bg.root
64  ```
65 
66 4. **How can I digitize a total of 100 sampled collisions merging background and signal hits for TPC sector 1?**
67  ```
68  o2-sim-digitizer-workflow -b --tpc-sectors=1 --simFile o2sim_bg.root --simFileS o2sim_sg.root -n 100
69  ```

71 # Missing things/Improvements to come
72 
73 At present the digitizer write individual digit files for each sector with names `tpc_digi_22_...`.
74 It is planned asap to make this more configurable and to outsource the writing to ROOT files in a different device.
75 
76 Configuration of the workflow via environment variables is going to be substituted via a proper mechanism once this
77 is implemented by DPL.
78 
79 Digitizers for other detectors shall be added.
80 
81 The polay distribution should be commicated via CDB or some init mechanism.
