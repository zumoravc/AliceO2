9 # MCH Mapping Interface
10 
11 Two APIs are offered to the MCH Mapping :
12 a [C interface](Interface/include/MCHMappingInterface/SegmentationCInterface.h)
13 (the core one) and a [C++ interface](Interface/include/MCHMappingInterface/Segmentation.h).
14 
15 The pattern of this dual interface follows the [Hourglass idea](https://github.com/CppCon/CppCon2014/tree/master/Presentations/Hourglass%20Interfaces%20for%20C%2B%2B%20APIs).
16 In a nutshell, the core interface is written in C so it can be accessed easily
17 from different languages and offers a stable ABI.
18 
19 But most users are only exposed to a header-only [C++
20 interface](Interface/include/MCHMappingInterface/Segmentation.h) which sits on
21 top of this core C interface.
22 
23 Details are to be found in the interface file or in the doxygen documentation,
24 but the basics of the interface is :
25 
26 ```c++
27 int detElemId{100};
28 
29 o2::mch::mapping::Segmentation seg{detElemId};
30 
31 double x{1.5};
32 double y{18.6};
33 
34 int b, nb;
35 book found = seg.findPadPairByPosition(x, y, b, nb);
36 
37 if (seg.isValid(b)) {
38 std::cout << "There is a bending pad at position " << x << "," << y << "\n"
39 << " which belongs to dualSampa " << seg.padDualSampaId(b)
40 << " and has a x-size of " << seg.padSizeX(b) << " cm\n";
41 }
42 
43 assert(b == seg.findPadByFEE(76, 9));
44 ```
45 
46 Note that cathode-specific segmentations can be retrieved from `Segmentation`
47 objects (using bending() and nonBending() methods) or created from scratch
48 using the `CathodeSegmentation(int detElemId, bool isBendingPlane)` constructor.

50 ## The segmentation() utility function
51 
52 To create segmentations and avoid duplicating objects, you can use the `segmentation`
53  function (mind the lower case first letter s).
54 
55 For instance :
56 
57 ```cpp
58 const Segmentation& seg1 = segmentation(501);
59 ...
60 (some code here)
61 ...
62 auto& seg2 = segmentation(501);
63 ```
64 
65 will result in only one object for segmentation 501 to be created.
66 
67 On the contrary, if you use :
68 
69 ```cpp
70 Segmentation seg1{501};
71 ...
72 ...
73 Segmentation seg2{501};
74 ```
75 
76 two separate objects will be created (and you will be hit twice by the
77 object creation time).
78 
79 > Note that depending on your use case, *both* approaches might be valid ones !
80 > While the factory one ensures you only get one object for each detection
81 > element, it creates all detection elements at once. So, if you only need the
82 > segmentation of one detection element, you'd better off *not* using the
83 > factory.

85 ## Implementations
86 
87 Currently two implementations ([Impl3](Impl3/README.md) and
88 [Impl4](Impl4/README.md), with Impl4 being the default one) are provided. Both
89 are faster than the legacy AliRoot one (but not by an order of magnitude...).
90 Extensive checks have been made to insure that Impl3 yields the same results as
91 AliRoot (see the `vsaliroot` directory in the [alo
92 repository](https://github.com/mrrtf/alo) ). The only difference between the
93 two implementations are the channel numbering. Impl3 uses Manu channel number,
94 aka Run1-2 numbering, while the Impl4 uses DualSampa channel numbering, aka
95 Run3 numbering. Impl3 is to be considered legacy and is used to deal with Run2
96 data only. It will be deprecated and removed at some point.
97 
98 Note that there is some (high) level of duplication between the two libraries,
99 but that's by design as the two libraries are meant to be completely
100 independent : by construction they can *not* be used at the same time, and at
101 some point in the future we'll remove Impl3 for instance. The only commonality
102 they have is that they implement the same interface, but they should not share
103 implementation details.

105 ## Contours
106 
107 Besides to core API (finding pads by position and by electronic ids), we offer
108 [convenience functions](SegContour/README.md) to compute the bounding box and [envelop of the segmentations](SegContour/include/MCHMappingSegContour/SegmentationContours.h).
109 
110 A simple executable, `o2-mch-mapping-svg-segmentation3` can be used to produce [SVG](https://developer.mozilla.org/en-US/docs/Web/SVG) plots
111 of the segmentations.
