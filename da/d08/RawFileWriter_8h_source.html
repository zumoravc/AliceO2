115  size_t pushBack(const char* ptr, size_t sz, bool keepLastOnFlash = true)
116  {
117  if (!sz) {
118  return buffer.size();
119  }
120  nBytesWritten += sz;
121  // do we have a space one this superpage?
122  if ((writer->mSuperPageSize - int(buffer.size())) < 0) { // need to flush
123  flushSuperPage(keepLastOnFlash);

125  auto offs = expandBufferBy(sz);

229  // This is a placeholder for the function responsible to split large payload to pieces
230  // fitting 8kB CRU pages.
231  // The RawFileWriter receives from the encoder the payload to format according to the CRU format
232  // In case this payload size does not fit into the CRU page (this may happen even if it is
233  // less than 8KB, since it might be added to already partially populated CRU page of the HBF)
234  // it will write on the page only part of the payload and carry over the rest on extra page(s).
235  // By default the RawFileWriter will simply chunk payload as is considers necessary, but some
236  // detectors want their CRU pages to be self-consistent and in case of payload splitting they
237  // add in the end of page to be closed and beginning of the new page to be opened
238  // (right after the RDH) detector-specific trailer and header respectively.
239  //
240  // The role of this method is to suggest to writer how to split the payload:
241  // If this method was set to the RawFileWriter using
242  // RawFileWriter::setCarryOverCallBack(pointer_on_the_converter_class);
243  // then the RawFileWriter will call it before splitting.
244  //
245  // It provides to the carryOverMethod method the following info:
246  // rdh : RDH of the CRU page being written
247  // data : original payload received by the RawFileWriter
248  // ptr : pointer on the data in the payload which was not yet added to the link CRU pages
249  // maxSize : maximum size (multiple of 16 bytes) of the bloc starting at ptr which it can
250  // accomodate at the current CRU page (i.e. what it would write by default)
251  // splitID : number of times this payload was already split, 0 at 1st call
252  // trailer : what it wants to add in the end of the CRU page where the data starting from ptr
253  // will be added. The trailer is supplied as an empy vector, which carryOverMethod
254  // may populate, but its size must be multiple of 16 bytes.
255  // header : what it wants to add right after the RDH of the new CRU page before the rest of
256  // the payload (starting at ptr+actualSize) will be written
257  //
258  // The method mast return actual size of the bloc which can be written (<=maxSize).
259  // If this method populates the trailer, it must ensure that it returns the actual size such that
260  // actualSize + trailer.size() <= maxSize
261  // In case returned actualSize == 0, current CRU page will be closed w/o adding anything, and new
262  // query of this method will be done on the new CRU page
263 
264  int carryOverMethod(const RDHAny*, const gsl::span<char> data, const char* ptr, int maxSize, int splitID,
265  std::vector<char>& trailer, std::vector<char>& header) const
266  {
267  return maxSize; // do nothing
268  }
