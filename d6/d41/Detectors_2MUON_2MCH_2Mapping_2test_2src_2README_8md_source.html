5 # MCH Mapping Tests
6 
7 ## Stress tests
8 
9 By default the unit (and thus short) tests are running without any input.
10 
11 To go further though, some stress tests take as input a JSON file containing test
12 positions, using the `--testpos` option (it is an option of the test module,
13 not of the test program itself, hence the `--` syntax) and call (part of) the
14 test like so :
15 
16 ```bash
17 o2-test-mchmapping-StressTest4 --run_test="*/*/*TestPos*" -- --testpos /some/dir/to/test_random_pos.json
18 ```
19 
20 ## Pad indices tests
21 
22 In the segmentation interface, the various findXXX method return an integer
23 that was first designed as a padId, that would be relevant only within a given
24 implementation. It was later decided to use this integer as a pad *index* (from
25 0 to number of pads in the cathode or in the full detection element), and to
26 guarantee that this pad index would be the same whatever the implementation is
27 (i.e. the implemeter has to ensure that this guarantee is fullfilled). The
28 `o2-test-mchmapping-PadIndices` test programs are meant for that. They take as
29 input test files that are generated by the
30 `o2-mchmapping-generate-pad-indices-implXXX` programs. See for instance the
31 `generate-pad-indices.sh` script. The generation program(s) generate by
32 default all the indices of a given detection element, unless the `--sparse`
33 option is used, in which case about a 1/16th of the pad indices are generated,
34 which should be enough to spot issues. For a new implementation it is
35 recommended though to test *without* the `--sparse` option. This is to
36 minimize the size of the generated json files.
37 
38 The number of indices generated can be checked using for instance :
39 
40  cat test_pad_indices_de100.json| jq . | grep padindex | wc -l
41 
42 which assumes the [jq](https://stedolan.github.io/jq/) utility program is
43 available.
