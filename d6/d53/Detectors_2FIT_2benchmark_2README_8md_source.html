5 # Documentation for Performance testing
6 This document will summarize the tools that can be used to get information about the memory and CPU time evolution of simulations in ALICE O2.
7 
8 In this folder you will find two scripts:
9 1. `monitor.sh`
10 2. `process.py`
11 
12 Both of these scripts define the two step procedure (**monitoring** and **processing**) of obtainging performance metrics of interest: _maximum memory, average memory, maximum CPU time, wall clock time, (CPU)/(wall clock) time ratio_ and lastly _plots of the evolving memory and cpu as a function of wall clock time._
13 
14 ## 1) Monitoring
15 You can monitor whatever you like as:
16 
17 `$> ./monitor.sh <your o2 command>`
18 
19 To obtain plots of (FairMQ) devices operating in the simulation you will have to generate a **logfile** as:
20 
21 `$> ./monitor.sh <your o2 command> | tee o2xxx.log`
22 
23 e.g. if you wish to monitor 50 pp (pythia) events with Geant3 as the VMC backend using the FIT detector and utilizing parallel mode with 2 simulation workers AND keep track of FairMQ devices, you can do:
24 
25 `$> ./monitor.sh o2-sim -g pythia8 -e TGeant3 -m FV0 FT0 FDD -j 2 -n 50 |Â tee o2sim.log`
26 
27 ---
28 
29 Similarly you can monitor the digitization routine as:
30 
31 `$> ./monitor.sh o2-sim-digitizer-workflow -b --run | tee o2digi.log`
32 
33 NB! notice the `--run` that is needed (only digitization) in order to overload the PIPE `|` command in DPL (Data Processing Layer).
34 
35 The `./monitor.sh` script will generate 4 .txt files in total: _mem_evolution_xxxx.txt, cpu_evolution_xxxx.txt, time_evolution_xxxx.txt, pid_evolution_xxxx.txt_. Here _xxxx_ is the PID (process identifcation) number of the main process responsible for the command (driver application). You will have to parse two of these files (mem and cpu) in the next step.
36 
37 ## 2) Processing
38 The monitored data has to be processed as:
39 `$> python3 process.py mem_evolution_xxxx.txt cpu_evolution_xxxx.txt`
40 
41 This will generate an output:
42 
43 ```Your command was: o2-sim -g pythia8 -e TGeant3 -m FV0 FT0 FDD -j 2 -n 50
44 You have monitored o2 simulation in parallel.
45 
46 ********************************
47 max mem: 723.30 MB
48 mean mem: 544.63 MB
49 max cpu: 120.69s
50 Total wall clock time: 82.54 s
51 Ratio (cpu time) / (wall clock time) : 1.46
52 ********************************
53 ```
