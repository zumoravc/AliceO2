template<typename T>
struct std::is_trivially_copyable< Point3D< T > >

Defining Point3D explicitly as trivially copyable

std::is_trivially_copyable<ROOT::Math::Cartesian3D<T>> fails because the class implements a copy constructor, although it does not much more than the default copy constructor. We need Point3D to fulfill the condition in order to make types inheriting from it or using it as member can be safely detected as messageable.

We believe that Point3D is messageable and explicitly specialize the type trait. There is a unit test for checking trivial copy This is a workaround, we will also make suggestions to fix the cause in ROOT itself TODO: delete once it is fixed in ROOT
