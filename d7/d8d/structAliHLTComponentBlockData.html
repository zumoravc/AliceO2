Detailed Description

This is the decription of data blocks exchanged between components. IMPORTANT: The validity of fPtr and fOffset is different for input and output blocks:

  • input blocks: The fPtr member always points to the beginning of the data of size fSize. fOffset is ignored and should be in most case 0.
  • output blocks: The fPtr member is ignored by the framework. fOffset must specify the start of the data relative to the output buffer. The data block has size fSize.
