Detailed Description

This is a simple FairMQ monitoring class assumption is the messages are O@ messages (constructed supported methods in O2Device). The "data" channel can be configured as any socket type, it will appropriately send requests, send/receive messages or send replies. All incoming traffic is dumped on screen in the form of a hex dump for both the header block and payload block.
