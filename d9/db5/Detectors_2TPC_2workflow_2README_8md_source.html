5 # DPL workflows for the TPC
6 
7 ## TPC reconstruction workflow
8 The TPC reconstruction workflow starts from the TPC digits, the *clusterer* reconstructs clusters in the
9 [ClusterHardware](../../../DataFormats/Detectors/TPC/include/DataFormatsTPC/ClusterHardware.h) format.
10 The clusters are directly written in the *RAW page* format. The raw data are passed onto the *decoder*
11 providing the [TPC native cluster](../../../DataFormats/Detectors/TPC/include/DataFormatsTPC/ClusterNative.h)
12 format to the *tracker*.
13 
14 Note: The format of the raw pages is preliminary and does not reflect what is currently implemented in the CRU.
15 
16 The workflow consists of the following DPL processors:
17 
18 * `tpc-digit-reader` -> using tool [o2::framework::RootTreeReader](../../../Framework/Utils/include/Utils/RootTreeReader.h)
19 * `tpc-clusterer` -> interfaces [o2::tpc::HwClusterer](../reconstruction/include/TPCReconstruction/HwClusterer.h)
20 * `tpc-cluster-decoder` -> interfaces [o2::tpc::HardwareClusterDecoder](../reconstruction/include/TPCReconstruction/HardwareClusterDecoder.h)
21 * `tpc-tracker` -> interfaces [o2::tpc::GPUCATracking](../reconstruction/include/TPCReconstruction/GPUCATracking.h)
22 * `tpc-track-writer` -> implements simple writing to ROOT file
23 
24 Depending on the input and output types the default workflow is extended by the following readers and writers:
25 * `tpc-raw-cluster-writer` writes the binary raw format data to binary branches in a ROOT file
26 * `tpc-raw-cluster-reader` reads data from binary branches of a ROOT file
27 * `tpc-cluster-writer` writes the binary native cluster data to binary branches in a ROOT file
28 * `tpc-cluster-reader` reads data from binary branches of a ROOT file
29 
30 MC labels are passed through the workflow along with the data objects and also written together with the
31 output at the configured stages (see output types).
32 
33 ### Input data
34 The input can be created by running the simulation (`o2sim`) and the digitizer workflow (`digitizer-workflow`).
35 The digitizer workflow produces the file `tpcdigits.root` by default, data is stored in separated branches for
36 all sectors.
37 
38 The workflow can be run starting from digits, raw clusters, or (native) clusters, or directly attached to the
39 `o2-sim-digitizer-workflow`, see comment on inputs types below.
40 
41 ### Quickstart running the reconstruction workflow
42 The workflow is implemented in the `o2-tpc-reco-workflow` executable.
43 
44 Display all options
45 ```
46 o2-tpc-reco-workflow --help
47 ```
48 
49 Important options for the `tpc-digit-reader` as initial publisher
50 ```
51 --infile arg Name of the input file
52 --treename arg (=o2sim) Name of the input tree
53 --digitbranch arg (=TPCDigit) Digit branch
54 --mcbranch arg (=TPCDigitMCTruth) MC label branch
55 --nevents Number of events
56 ```
57 
58 The `tpc-raw-cluster-reader` uses the same options except the branch name configuration
59 --databranch arg (==TPCClusterHw) RAW Cluster branch
60 --mcbranch arg (==TPCClusterHwMCTruth) MC label branch
61 
62 Options for the `tpc-track-writer` process
63 ```
64 --outfile arg (=tpctracks.root) Name of the output file
65 --treename arg (=tpcrec) Name of tree
66 --nevents arg (=-1) Number of events to execute
67 --terminate arg (=process) Terminate the 'process' or 'workflow'
68 --track-branch-name arg (=TPCTracks) configurable branch name for the tracks
69 --trackmc-branch-name arg (=TPCTracksMCTruth) configurable branch name for the MC labels
70 ```
71 
72 Examples:
73 ```
74 o2-tpc-reco-workflow --infile tpcdigits.root --tpc-sectors 0-17 --tracker-options "cont refX=83 bz=-5.0068597793"
75 ```
76 
77 ```
78 o2-tpc-reco-workflow --infile tpcdigits.root --tpc-sectors 0-17 --disable-mc 1 --tracker-options "cont refX=83 bz=-5.0068597793"
79 ```

81 ### Global workflow options:
82 ```
83 --input-type arg (=digits) digitizer, digits, raw, clusters
84 --output-type arg (=tracks) digits, raw, clusters, tracks
85 --disable-mc arg (=0) disable sending of MC information
86 --tpc-lanes arg (=1) number of parallel lanes up to the tracker
87 --tpc-sectors arg (=0-35) TPC sector range, e.g. 5-7,8,9
88 ```
89 
90 #### Input Type
91 Input type `digitizer` will create the clusterers with dangling input, this is used
92 to connect the reconstruction workflow directly to the digitizer workflow.
93 
94 All other input types will create a publisher process reading data from branches of
95 a ROOT file. File and branch names are configurable. The MC labels are always read
96 from a parallel branch, the sequence of data and MC objects is assumed to be identical.
97 
98 #### Output Type
99 The output type selects up to which final product the workflow is executed. Multiple outputs
100 are supported in order to write data at intermediate steps, e.g.
101 ```
102 --output-type raw,tracks
103 ```
104 
105 MC label data are stored in corresponding branches per sector. The sequence of MC objects must match
106 the sequence of data objects.
107 
108 By default, all data is written to ROOT files, even the data in binary format like the raw data and cluster
109 data. This allows to record multiple sets (i.e. timeframes/events) in one file alongside with the MC labels.
110 
111 #### Parallel processing
112 Parallel processing is controlled by the option `--tpc-lanes n`. The digit reader will fan out to n processing
113 lanes, each with clusterer, and decoder. The tracker will fan in from multiple parallel lanes.
114 For each sector, a dedicated DPL data channel is created. The channels are distributed among the lanes.
115 The default configuration processes sector data belonging together in the same time slice, but in earlier
116 implementations the sector data was distributed among multiple time slices (thus abusing the DPL time
117 slice concept). The tracker spec implements optional buffering of the input data, if the set is not complete
118 within one invocation of the processing function.
119 
120 #### TPC sector selection
121 By default, all TPC sectors are processed by the workflow, option `--tpc-sectors` reduces this to a subset.

125 #### TPC CA tracker
126 The [tracker spec](src/CATrackerSpec.cxx) interfaces the [o2::tpc::GPUCATracking](../reconstruction/include/TPCReconstruction/GPUCATracking.h)
127 worker class which can be initialized using an option string. The processor spec defines the option `--tracker-option`. Currently, the tracker
128 should be run with options:
129 ```
130 --tracker-option "refX=83 cont"
131 ```
132 
133 The most important tracker options are:
134 ```
135 cont activation of continuous mode
136 refX= reference x coordinate, tracker tries to propagate all track to the reference
137 bz= magnetic field
138 ```
