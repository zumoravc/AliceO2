5 # MCH Raw Data Decoder
6 
7 There are currently two data formats to be dealt with : the "bare" one coming
8 out of the CRU when no user logic (UL) is used, or the "UL" one where user
9 logic is actually used. Each of the two formats can be used in charge sum (the
10 sampa electronics does the charge integration) or in sample mode (the sampa
11 electronics transmit all the charge samples). All formats are composed of
12 {RawDataHeader,Payload} pairs.
13 
14 The work of the MCH Raw Data decoder is to decode one single such pair, also
15 called a (CRU) page. How the loop on the raw pages is done is *not* the decoder business.
16 
17 ## createPageDecoder
18 
19 On the reading/consumer/decoding end, the choice of the internal decoder to use is made
20 using the data itself. You must give (at least) a part of a raw data buffer that contains
21  a (valid) RawDataHeader. That RDH is used to deduce which implementation is picked.
22 
23  gsl::span<const std::byte> rawbuffer = ... ;
24  auto pageDecoder = o2::mch::raw::createPageDecoder(rawbuffer,channelHandler);
25 
26 The `pageDecoder` that is returned is a function that you have to call on each
27 data page that you want to decode :
28 
29  while(some_data_is_available) {
30  // get some memory buffer from somewhere ...
31  buffer = ...
32 
33  // decode that buffer
34  pageDecode(buffer);
35  }
36 
37 Internally the implementation is using templatized implementation, `PageDecoderImpl<FORMAT,CHARGESUM>`.
38 Currently the following template parameters combinations have been tested :
39 
40 | FORMAT | CHARGESUM |
41 | :-------------: | :-----------: |
42 | BareFormat | ChargeSumMode |
43 | BareFormat | SampleMode |
44 | UserLogicFormat | ChargeSumMode |
45 | UserLogicFormat | SampleMode |
46 
47 The `createPageDecoder` function requires two parameters : a raw memory buffer
48 (in the form of a `gsl::span<const std::byte>` (note that the span is on constant bytes,
49 i.e. the input buffer is read-only)
50 and a `SampaChannelHandler`.
51 
52 ## SampaChannelHandler
53 
54 The `SampaChannelHandler` is a function, that takes a dual sampa
55 identifier (in electronics realm, aka solar,group,index tuple), a channel
56 identifier within that dual sampa, a `SampaCluster` and returns nothing, i.e. :
57 
58 > A word of caution here about the naming. A `SampaCluster` is a group of raw
59 > data samples of *one* dual sampa channel, and has nothing to do with a
60 > cluster of pads or something alike.
61 
62 ```.cpp
63 using SampaChannelHandler = std::function<void(DsElecId dsId,
64  uint8_t channel,
65  SampaCluster)>;
66 ```
67 
68 That function is called by the raw data decoder for each `SampaCluster` that it
69 finds in the data.
70 A very simple example would be a function to dump the SampaClusters :
71 
72 ```.cpp
73 SampaChannelHandler handlePacket(DsElecId dsId, uint8_t channel, SampaCluster sc) {
74  std::cout << fmt::format("{}-ch-{}-ts-{}-q-{}", asString(dsId), channel, sc.timestamp, sc.chargeSum));
75 }
76 // (note that this particular function only correctly handles the SampaCluster in ChargeSum Mode)
77 ```
