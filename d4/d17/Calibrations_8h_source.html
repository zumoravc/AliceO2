16 // TRD calibration class for parameters which are saved frequently(/run) //
17 // 2019 - Ported from various bits of AliRoot (SHTM) //
18 // A wrapper api class to provide a user intuitive interface to the
19 // underlying calibrations classes and ccdb and make life easy for all
20 // bold statement, lets see if we get there.
21 //
22 // Questions :
23 // 1. Its not clear to me yet what happens if I query an object that
24 // is not defined as input in the dpl for the device we are using.
25 //
26 // This pulls in functions from various subordinate classes like CalPad CalROC
27 // CalDet etc. and renames them to clearly reflect their applicability,
28 //
29 // This class does *not* store pointers to any subordinate objects as that
30 // would imply caching. At each request the underlying object stored in
31 // CCDB is sought, either directly or for that which is provided by DPL. The
32 // latter still being unclear (17/09/2019)
