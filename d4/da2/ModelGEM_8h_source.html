15 // ================================================================================
16 // How are the electron efficiencies obtained?
17 // ================================================================================
18 // Within the scope of two PhD thesis models have been derived in order to
19 // describe the collection as well as the extraction efficiencies for GEM
20 // foils. In the following you can find a brief sketch about the procedure behind this.
21 // Details can be found in the following two papers which emerged out of this
22 // research:
23 // [1] Paper Jonathan,
24 // [2] Paper Viktor.
25 //
26 // Simulations / Measurements [1]:
27 // For different GEM geometries (standard, medium, large pitch) the electric potentials
28 // and fieldmaps were obtained by numerical calculations in Ansys. The fieldmaps were
29 // then used in Garfield++ in order to simulate the drift of the charge carriers
30 // and the amplification processes. In order to obtain the efficiencies a fixed
31 // amount of electrons has been randomely distributed above the GEM for the simulations.
32 // The efficiencies were thereupon derived by counting where the initial electrons and
33 // electrons from the amplification region ended, e.g. Copper top / bottom, anode etc.
34 // Indeed the simulated efficiencies are in a good agreement to measurements. See [1] for
35 // more details.
36 //
37 // Calculations [2]:
38 // In oder to get an analytic understanding of the efficiencies a simplified and
39 // two-dimensional model has been investigated. Simplified means: No differentian
40 // between an inner and an outer diameter, no Polyimide layer, no gas/no diffusion
41 // and two-dimensional cut of the hexagonal 3D GEM structure.
42 // The resulting equations are in a good agreement to the results from the simulations
43 // in terms of limits, offsets and curves for different pitches (in case of no diffusion).
44 // Nevertheless differences can be found due to the simplifications in the model.
45 // By introducing three fit parameter (s1, s2 and s3) the calculations can be tuned
46 // in a way to describe the simulated datapoints. The resulting equations
47 // describe the efficiencies in a full region and for different GEM geometries.
48 //
49 // The results from the fitted equations are implemented in this class.
50 
51 // ================================================================================
52 // Remarks for naming of the variables:
53 // ================================================================================
54 //
55 // mFitElecEffNumberHoles (in PhD/paper: N)
56 // Describes the number of holes for the 2D model calculations. The number
57 // of holes is given by 2N-1, i.e. mFitElecEffNumberHoles=2 refers to 3 GEM holes (one
58 // central GEM hole and two GEM holes at the outside).
59 //
60 // mFitElecEffThickness (in PhD/paper: d) [unit: micrometers]
61 // Thickness of the GEM foil which
62 //
63 // mFitElecEffPitch (in PhD/paper: p) [unit: micrometers]
64 // Pitch of the GEM foil.
65 //
66 // mFitElecEffHoleDiameter (in PhD/paper: L) [unit: micrometers]
67 // Hole diameter for the GEM hole. There is no differentiation between an inner
68 // and an outer hole diameter for the 2D model calculations.
69 //
70 // mFitElecEffWidth (in PhD/paper: w) [unit: micrometers]
71 // Describes the width for a unit cell (pitch) + 2x the distance to the end of the
72 // GEM electrodes. This variable can be expressed in terms of the pitch and the hole
73 // diameter according to w=2p-L. It is only used for internal calculations and no
74 // definition is required by the user.
75 //
76 // mFitElecEffDistancePrevStage (in PhD/paper: g1) [unit: micrometers]
77 // Here g1/2 describes the distance from the center of the GEM foil to the cathode
78 // or the previous amplification stage.
79 //
80 // mFitElecEffDistanceNextStage (in PhD/paper: g2) [unit: micrometers]
81 // Here g2/2 describes the distance from the center of the GEM foil to the anode
82 // or the next amplification stage.
83 //
84 // mGeometryTuneEta1 (in PhD/paper: s1) [unitless]
85 // This is a fit parameter which has been used in order to scale eta1 for the fit
86 // of the calculations to the simulations.
87 //
88 // mGeometryTuneEta2 (in PhD/paper: s2) [unitless]
89 // This is a fit parameter which has been used in order to scale eta2 for the fit
90 // of the calculations to the simulations.
91 //
92 // mGeometryTuneDiffusion (in PhD/paper: s3) [unitless]
93 // This is a fit parameter which has been used in order to tune the equations to
94 // describe diffusion.
