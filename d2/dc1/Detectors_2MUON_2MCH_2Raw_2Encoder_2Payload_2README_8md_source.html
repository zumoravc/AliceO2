5 The `EncoderPayload` library is building raw data buffer for *data payloads*.
6 How those payloads are then ordered exactly in memory / on file is not the
7 library business. This choice has been made to minimize the dependency of the
8 Encoder code on the RawDataHeader itself. To that end the Encoder is producing
9 data buffers consisting of (header,payload)
10 [blocks](/include/MCHRawEncoderPayload/DataBlock.h), where header is *not* a
11 RDH but a simpler (and non versioned) struct. Contrary to (RDH,payload) blocks,
12 the payload can be any size (while in RDH the size is limited to 65536 bytes).
13 The part of the detector that a payload block references is identified through
14 the `solarId` field.
15 
16 ## createPayloadEncoder
17 
18 To creation of a MCH PayloadEncoder is handled through a template function.
19 For instance, to create an Encoder that builds raw data in BareFormat and
20 ChargeSumMode, use :
21 
22 ```.cpp
23 #include "MCHRawEncoderPayload/PayloadEncoder.h"
24 
25 auto encoder = o2::mch::raw::createPayloadEncoder<BareFormat,ChargeSumMode>();
26 
27 ```
28 
29 Then for each interaction record (aka orbit,bunch crossing pair) you tell
30 the encoder to start a new Heartbeat Frame, fill it with all the data
31 of the channels and then extract the memory buffer.
32 
33 ```.cpp
34 vector<uint8_t> buffer;
35 for ( loop over interactions )
36  encoder->startHeartbeatFrame(orbit,bc);
37  for ( loop over digits ) {
38  DsElecId elecId = ... // get electronic dual sampa id from digit
39  int sampaChannelId = ... // get sampa channel id from digits
40  vector<SampaClusters> clusters = ... // create sampa clusters from digits
41  encoder->addChannelData(elecId,sampaChannelId,clusters);
42  }
43 encoder->moveToBuffer(buffer);
44 }
45 ```
46 
47 At this stage the buffer contains (header,payload) blocks for all the
48 interactions you looped over. But it is *not* yet a valid raw data, as it lacks
49 RDHs (and has mch-specific headers instead).
50 
51 The conversion from payload buffers to RDH-base buffers is basically done using
52 the `o2::mch::raw::PayloadPaginator` class (internally using the `o2::raw::RawFileWriter` framework
53 class). A typical usage example is in the [digit2raw](Digit/) program.
54 
55 ## Advanced usage
56 
57 The `createPayloadEncoder` actually takes one parameter, a function that
58 converts a solarId into a FeeLinkId (aka a Solar2FeeLinkMapper). The default
59 value should fit the electronic mapping of the actual detector and thus should
60 be fine in most cases. For special use cases a different mapper can be provided
61 though.
